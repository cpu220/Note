网络七层协议

物理层（Physical Layer）：Ethernet、USB、Wi-Fi、蓝牙、光纤等。

数据链路层（Data Link Layer）：Ethernet、PPP、HDLC、MAC（媒体访问控制协议）等。

网络层（Network Layer）：IPv4、IPv6、ICMP（Internet控制消息协议）、OSPF（开放最短路径优先协议）、BGP（边界网关协议）等。

传输层（Transport Layer）：TCP（传输控制协议）、UDP（用户数据报协议）等。

会话层（Session Layer）：NetBIOS（网络基本输入/输出系统）、RPC（远程过程调用）等。

表示层（Presentation Layer）：SSL/TLS（安全套接层/传输层安全）、ASCII码、JPEG（联合图像专家组）等。

应用层（Application Layer）：HTTP（超文本传输协议）、FTP（文件传输协议）、SMTP（简单邮件传输协议）、POP3（邮局协议3）、IMAP（互联网邮件访问协议）、DNS（域名系统）、SSH（安全外壳协议）等。

# 会话层 Session
负责建立和管理应用程序之间的通信会话，包括对多个双向消息的控制和管理，以便在只完成连续消息的一部分时，可以通知应用，从而使表示层看到的数据是连续的。
OSI七层模型中的第五层，主要功能室创建、管理、和维护会话。
会话层还可以提供一些增强的服务，如认证，权限，会话，回复，同步等。

* 网关

# 传输层 Transport
负责向两个主机中，进程之间的通信提供服务，包括是否选择差错回复协议还是无差错回复协议，及在同一主机上对不同应用的数据流的输入进行复用，还包括对受到的顺序不对的数据包的重新排序
## 典型设备
网关
## 数据单元
TCP段或UDP报文，它是由会话层产生的数据经过传输层处理后的结果，包含了端口号和校验和等信息。


## 协议
* TCP (Transmission Control Protocol )
    * 传输控制协议提供可靠的面向连接的服务，传输数据前须先建立链接，结束后释放。可靠的全双工信道。可靠、有序、无丢失、不重复
* UDP (User Datagram Protocol)
    * 用户数据报协议提供不可靠的无连接服务，数据传输前不需要建立链接，结束后也不释放。不可靠的全双工信道。不可靠、无序、有丢失、有重复
* DCCP (Datagram Congestion Control Protocol)
    * 数据报拥塞控制协议，是一种不可靠的传输层协议，它提供了与TCP类似的可靠性保证，但是它是面向消息的，而不是面向字节流的。DCCP是基于UDP的，它使用了UDP的复用机制，因此可以在同一个端口上同时提供多个DCCP连接。DCCP的可靠性保证是通过在DCCP头部中使用序列号和确认应答机制来实现的。DCCP还提供了拥塞控制机制，这使得DCCP可以在拥塞的网络中提供可靠的传输服务。
* SCTP (Stream Control Transmission Protocol)
    * 流控制传输协议，是一种面向消息的传输层协议，它提供了与TCP类似的可靠性保证，但是它是面向消息的，而不是面向字节流的。SCTP是基于UDP的，它使用了UDP的复用机制，因此可以在同一个端口上同时提供多个SCTP连接。SCTP的可靠性保证是通过在SCTP头部中使用序列号和确认应答机制来实现的。SCTP还提供了拥塞控制机制，这使得SCTP可以在拥塞的网络中提供可靠的传输服务。
* RTP (Real-time Transport Protocol)
    * 实时传输协议，是一个网络传输协议，它是由IETF的多媒体传输工作小组1996年在RFC 1889中公布的。它被设计用来在不可靠的网络中传输多媒体数据，它通常用于流媒体系统。RTP本身并不提供数据传输服务，它通常运行于UDP之上，使用UDP的复用机制，因此可以在同一个端口上同时提供多个RTP连接。RTP的可靠性保证是通过在RTP头部中使用序列号和确认应答机制来实现的。RTP还提供了拥塞控制机制，这使得RTP可以在拥塞的网络中提供可靠的传输服务。
* PPTP (Point-to-Point Tunneling Protocol)
    * 点对点隧道协议，是一种网络传输协议，它是由微软公司和其它厂商联合开发的，它是一种VPN协议，它可以在不可靠的网络中传输多媒体数据，它通常用于VPN系统。PPTP本身并不提供数据传输服务，它通常运行于TCP之上，使用TCP的复用机制，因此可以在同一个端口上同时提供多个PPTP连接。PPTP的可靠性保证是通过在PPTP头部中使用序列号和确认应答机制来实现的。PPTP还提供了拥塞控制机制，这使得PPTP可以在拥塞的网络中提供可靠的传输服务。
* TLS (Transport Layer Security)
    * 传输层安全协议，是一种面向连接的协议，它是一种无状态的协议，它不保留任何发送数据的信息。TLS协议的主要任务是在两个主机之间传输数据，它定义了两个主机之间传输数据的规范。TLS协议的主要特点是它是一种无连接的、不可靠的、无状态的协议。TLS协议的无连接性是指在发送分组之前，不需要建立连接。TLS协议的不可靠性是指它不能保证分组一定能够到达目的主机，也不能保证分组到达目的主机的顺序和发送的顺序相同。TLS协议的无状态性是指它不保留任何发送数据的信息。
